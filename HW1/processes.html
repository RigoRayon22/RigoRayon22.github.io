What is a Process?

A process is a running instance of a program. Besides the code, it includes its current CPU state (registers), memory mappings, open files, and permissions.

The OS keeps track of processes using a data structure often called a process control block (PCB).

Scheduling

Scheduling is the OS decision-making about which process runs next. Goals often include keeping the CPU busy, making interactive apps feel snappy, and being fair across tasks.

  • Time slice: a small time window each process can run.
  • Preemption: stopping a process when its slice ends.
  • Context switch: saving one process state and loading another.

System Calls (How programs ask for OS help)

Programs use system calls to request protected services like reading a file or creating a new process. Under the hood, a system call triggers a controlled transition into kernel mode, the OS does the work, and then returns to the app.
